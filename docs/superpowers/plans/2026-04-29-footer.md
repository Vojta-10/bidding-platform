# Footer Component Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a full `Footer` and a slim `FooterSlim` component, wire them into public and auth pages respectively, and leave the dashboard footer-free.

**Architecture:** Two Server Components in `components/layout/footer/`. Public pages are moved into a `(public)` route group with a layout that wraps Navbar + content + Footer. Auth pages get individual layouts with FooterSlim. Dashboard layout stays unchanged.

**Tech Stack:** Next.js 16 App Router, Tailwind CSS v4, `font-heading` (Playfair Display), `font-sans` (Jakarta Sans), shadcn/ui color tokens

### Task 1: Create the full Footer component

**Files:**
- Create: `components/layout/footer/Footer.tsx`

- [ ] **Step 1: Create the file**

```tsx
// components/layout/footer/Footer.tsx
import Link from 'next/link';

const exploreLinks = [
  { label: 'Browse', href: '/auctions' },

  { label: 'Ending Soon', href: '/auctions?filter=ending' },

  return (
    <footer className='border-t bg-muted'>

            <p className='text-sm font-semibold text-foreground'>Explore</p>
            <ul className='flex flex-col gap-2'>
              {exploreLinks.map(({ label, href }) => (
                <li key={label}>
                  <Link
                    href={href}

                    {label}

            <p className='text-sm font-semibold text-foreground'>Company</p>

                  <Link
                    href={href}

        <div className='border-t border-border pb-8 pt-6'>
          <p className='text-xs text-muted-foreground'>
```

### Task 3: Integrate Footer into public pages via route group

A `(public)` route group in Next.js creates a shared layout for grouped pages without affecting URLs — `/` stays `/`. This layout adds Navbar + Footer around public content, while the dashboard layout (which has its own `app/dashboard/layout.tsx`) is unaffected.

**Files:**
- Create: `app/(public)/layout.tsx`
- Move: `app/page.tsx` → `app/(public)/page.tsx`

- [ ] **Step 1: Create the public layout**

```tsx
// app/(public)/layout.tsx
import Navbar from '@/components/layout/navigation/Navbar';
import Footer from '@/components/layout/footer/Footer';

export default function PublicLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <>
      <Navbar />
      <main className='flex-1'>{children}</main>
      <Footer />
    </>
  );
}
```

- [ ] **Step 2: Move the home page into the route group**

Route groups don't change URLs — `app/(public)/page.tsx` still serves `/`.

```bash
mkdir -p "app/(public)"
mv app/page.tsx "app/(public)/page.tsx"
```

- [ ] **Step 3: Start the dev server and verify**

```bash
npm run dev
```

- Open `http://localhost:3000` — expect Navbar at top, home page content in the middle, Footer with muted background panel at the bottom.
- Open `http://localhost:3000/dashboard` — expect Navbar, dashboard content, **no footer**.

- [ ] **Step 4: Commit**

```bash
git add "app/(public)/layout.tsx" "app/(public)/page.tsx"
git rm app/page.tsx
git commit -m "feat: add public layout with Footer, move home page into (public) route group"
```

### Task 4: Add FooterSlim to auth pages

The auth pages currently use `min-h-screen` on their page wrapper to center the form. Adding a layout that includes FooterSlim requires adjusting that — the root `<body>` is already `min-h-full flex flex-col`, so giving `<main>` a `flex-1` class fills the available height and the slim footer sits naturally at the bottom.

**Files:**
- Create: `app/sign-in/layout.tsx`
- Create: `app/sign-up/layout.tsx`
- Modify: `app/sign-in/page.tsx`
- Modify: `app/sign-up/page.tsx`

- [ ] **Step 1: Create the sign-in layout**

```tsx
// app/sign-in/layout.tsx
import FooterSlim from '@/components/layout/footer/FooterSlim';

export default function SignInLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <>
      <main className='flex flex-1 items-center justify-center p-4'>
        {children}
      </main>
      <FooterSlim />
    </>
  );
}
```
